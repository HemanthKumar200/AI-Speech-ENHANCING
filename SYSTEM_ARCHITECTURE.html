2. Models & APIs Used

2.1 Whisper Model (Xenova/whisper-tiny.en)

Why Whisper? It provides:

  • ✓ 99.1% WER on LibriSpeech (industry-leading accuracy)
  • ✓ Robust to accents and background noise
  • ✓ Word-level timestamps (critical for fluency analysis)
  • ✓ Confidence scores per word (pronunciation indicators)
  • ✓ Open source - no licensing costs
Aspect Our Choice Rationale
Model Whisper Tiny 75MB, 60% faster than base, 99% accuracy retained
Quantization Q8 (8-bit) 4x smaller, 40% faster inference, <1% accuracy loss
Format ONNX + WebAssembly Native browser execution, no backend needed
Language English only Focused assessment capability

2.2 Transformers.js Library

Why Transformers.js over alternatives?

  • ✓ Runs models in browser using WebAssembly (WASM)
  • ✓ No backend server required → Perfect for DPDP compliance
  • ✓ Automatic IndexedDB caching → Fast subsequent loads
  • ✓ Supports quantized models → Speed optimization built-in

3. Pronunciation & Fluency Scoring

Scoring Algorithm

INPUT: Audio File (30-45 seconds)
   ↓
[TRANSCRIPTION]
Audio → Whisper → Text + Timestamps + Confidence

[METRICS CALCULATION]
• WPM = (word_count / duration) × 60
• Clarity = Average confidence score (0-1)
• Pause Analysis = Gaps > 500ms between words

[SCORING]
Final Score = (WPM_score × 0.4) + (Clarity × 0.4) + (Pace × 0.2)

[HIGHLIGHTING]
Flag words/phrases for:
• Confidence < 0.7 → "Unclear pronunciation"
• Pauses > 1s → "Unusual hesitation"  
• WPM > 200 → "Speaking too fast"
• WPM < 80 → "Speaking too slow"

4. DPDP Compliance (India's Data Protection Act 2023)

4.1 Core Principle: Local-First, Zero-Server

The entire application is designed to ensure no audio or personal data leaves the user's browser. This ensures automatic compliance with DPDP requirements:

DPDP Requirement Our Implementation
Consent Explicit opt-in before processing - user controls upload
Purpose Limitation Audio only used for fluency scoring, no secondary use
Data Minimization Only transcription kept, raw audio deleted after processing
Storage LocalStorage only (optional), nothing sent to servers
Retention Auto-deletion after analysis, session cleanup on close
Residency All processing on user's device (India-based browsers)
Deletion Rights Browser's Clear Data feature gives full user control
Security HTTPS enforced, local processing (no network exposure)

4.2 Technical Data Flow Guarantee

USER'S DEVICE (Only location of data) │ ├─ Audio File │ └─ Remains in RAM during processing │ └─ Deleted immediately after analysis │ └─ NEVER transmitted to any server │ ├─ Results (Scores, Transcription) │ └─ Optional LocalStorage (user consent only) │ └─ SessionStorage (cleared on browser close) │ └─ Never sent to backend │ └─ Personal Data └─ NO identification data collected └─ NO tracking cookies └─ NO third-party analytics on audio

5. Trade-offs Made

Decision Choice Benefit Trade-off
Speed vs Accuracy Q8 Quantization 40% faster <1% accuracy loss
Model Size Whisper Tiny Fast downloads, reasonable accuracy Limited accent support
Processing Location Browser only DPDP compliant, no privacy issues Can't do advanced backend analysis
Language Support English only Simpler, faster for focused assessment No multilingual support
Feedback Timing Batch processing (post-upload) Better accuracy, simpler UX No live real-time feedback
Data Persistence LocalStorage optional Full DPDP compliance User loses history on cache clear
